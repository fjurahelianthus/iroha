# IROHA 設計總結

## 實作

- 語言：Python
- Parser：lark（LALR模式）
- 目標：interpreter

---

## 字符集分層

| 字符類型 | 角色 |
|----------|------|
| 漢字 | 保留類型名、builtin名、identifier的一部分 |
| 平假名 | 語法關鍵字和助詞，全部reserved |
| 片假名 | identifier的一部分（可與漢字混寫） |
| 全角符號 | 語法符號（（）「」。、） |

Identifier規則：漢字和片假名的任意組合，longest match。純漢字查保留字列表。函數identifier必須以`する/す`結尾。

## 類型系統

| 類型名 | 意義 |
|--------|------|
| `数` | 整數 |
| `浮` | 浮點數 |
| `符` | 字符或字串 |
| `判` | Boolean（值為`是`/`非`） |
| `数々` | 一維整數陣列 |
| `数々々` | 二維整數陣列（々可無限疊加） |

## Binding

```
xは数の1です。              # 顯式類型binding
xは数です。                 # 宣告不賦值
xは常に数の1です。          # 常數
```

---

## 函數

```
# 宣告
計算するの法は数のa、数のbと数のcを使え。

# 定義
計算するとは、
aとbをたすと答える。
終わり。

# 呼叫
aとbを計算する。
a、bとcを計算する。
```

> ⚠️ `a、b、cを計算する。` → warning: もう噛んじゃう

## 控制流

```
# 範圍迴圈
1から10まで[2ずつ]、
<body>
繰り返す。

xは是まで、
<body>

<elif>] # 可以有任意多個elif
```

## 內置動詞

| 動詞 | 意義 |
|------|------|
| `と言う` | print（複合token） |
| `と答える` | return（複合token） |
| `をたす` | 加法 |
| `をへる` | 減法 |
| `をかける` | 乗法 |
| `をわる` | 除法 |
| `をみる` | debug inspect |
| `をきく` | 讀取輸入 |

## 運算子

**比較：**

- `xはyより` → x > y
- `xはyより等` → x >= y
- x < y → 交換xy位置

**邏輯/Bitwise：**

| 符號 | 意義 |
|------|------|
| `且` | AND |
| `又` | OR |
| `の逆` | NOT（後置） |
| `異` | XOR |
| `動` | shift |

**Grouping：** `（x且y）の逆`

**優先級：** `動` > `且` > `又` > `異` > `の逆`

## Lexer 特殊處理

- `と言う`、`と答える` → 整體tokenize為複合token，消除と的歧義
- 平假名序列 → 直接tokenize為關鍵字
- 漢字片假名混寫 → longest match為IDENTIFIER
- する/す結尾 → FUNC_IDENTIFIER
- `「`與`」`之間的內容 → STRING_LITERAL，允許任意字符（包括全角符號），不進行escape處理
- 數字與一元負號的tokenize → ASCII數字與半角符號，與現代語言一致

## 報錯風格

```
エラー：xは知らない。       # 未定義變數
エラー：これは違います。     # 類型錯誤
エラー：だめです。           # 非法操作
警告：もう噛んじゃう         # 全頓號分隔參數
```
